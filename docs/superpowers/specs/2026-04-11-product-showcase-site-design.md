# 产品展示型网站设计说明

## 1. 项目概述

本项目是一个面向潜在客户的响应式产品展示型网站，核心目标是通过清晰、可信、具有视觉吸引力的产品内容，帮助客户快速了解产品特点与优势，建立初步信任，并引导客户通过电话、微信或留言表单主动联系。

网站不集成支付系统，不涉及在线交易闭环，重点放在产品展示、品牌背书和线索收集上。这样既降低开发复杂度，也更符合农产品销售中需要灵活沟通、按需求报价和建立长期客户关系的业务模式。

## 2. 目标与边界

### 2.1 核心目标

- 通过首页视觉化展示快速传递主营产品与品牌气质。
- 通过分类页和详情页完整展示产品图片、规格、包装和卖点。
- 通过关于我们页面强化经营历史、产地环境与生产经验，增强信任感。
- 通过电话、微信、询盘表单和留言表单收集客户意向，促进后续成交。

### 2.2 非目标

- 不实现在线支付、订单、物流、会员或库存管理。
- 不实现真实后端服务，当前阶段的留言和询盘只做前端演示闭环。
- 不实现复杂的后台权限系统，留言管理页仅作为前端演示功能。

## 3. 用户与使用场景

### 3.1 目标用户

- 对茶叶、椴木黑木耳及其他农产品有采购需求的个人客户。
- 寻找稳定农产品供应的渠道客户、礼品客户或熟人转介绍客户。
- 首次接触品牌、需要通过网页快速判断可信度的潜在客户。

### 3.2 核心场景

- 用户通过手机查看首页，快速了解主打产品并点击电话或微信咨询。
- 用户浏览具体品类，比较不同采摘时间的绿茶或不同季节的黑木耳。
- 用户在产品详情页查看高清图片、规格和包装后提交询盘。
- 用户阅读关于我们页面，基于产地信息和经营历史建立初步信任。

## 4. 设计方向

网站整体采用“混合型产品展示站”定位，兼顾产地故事、产品品质和咨询转化。风格上强调自然、可信、朴实中带品质感，避免过度商业化和过于花哨的动画表达。

### 4.1 视觉关键词

- 自然
- 可信
- 朴实
- 精致

### 4.2 色彩建议

- 茶绿：用于品牌主色、按钮强调和自然氛围塑造。
- 米白：用于页面底色，提升清爽感和阅读舒适度。
- 木色：用于辅助卡片和包装展示区域，传达农产品质感。
- 深棕：用于标题、重点文案和局部对比。

## 5. 信息架构

网站采用品牌展示型单站结构，包含 6 个核心页面或视图：

1. 首页
2. 产品分类页
3. 产品详情页
4. 关于我们页
5. 联系我们页
6. 留言管理页

### 5.1 导航结构

顶部主导航建议如下：

- 首页
- 产品中心
- 关于我们
- 联系我们
- 留言管理

首页内部提供产品分类导航，作为二级入口直达对应品类。所有产品卡片和详情页统一提供“立即咨询”入口，避免用户在转化路径中迷失。

## 6. 页面设计

### 6.1 首页

首页采用转化优先的长页面结构，自上而下依次完成吸引、介绍、建立信任和引导咨询。

包含以下模块：

1. 轮播图区域
   - 主推产品为茶叶，次推产品包括椴木黑木耳、葛根粉、橡子粉、茶油、农家土鸡蛋。
   - 每张轮播图包含产品图、简短卖点文案、“查看产品”和“立即咨询”按钮。
   - 支持自动轮播与手动切换。

2. 产品分类导航区
   - 展示所有一级分类入口。
   - 绿茶展示“清明前、清明、谷雨”等标签。
   - 椴木黑木耳展示“春耳、冬耳”等标签。

3. 热门产品区
   - 展示 3 到 5 款主打产品。
   - 卡片包含产品图、名称、简介、规格摘要、包装缩略图和快速联系按钮。
   - 价格区域保留位置，但默认隐藏。

4. 产品优势与包装展示区
   - 展示产地、经验、品质、传统工艺等核心卖点。
   - 展示茶叶包装、木耳包装等包装图片，增强专业感和礼品感。

5. 店铺介绍区
   - 突出“做茶 40 余年”“种植木耳 40 余年”等经营历史。
   - 展示产地环境、采摘和生产过程等图文内容。

6. 联系方式区
   - 清晰展示电话、微信和咨询引导文案。
   - 提供一键拨号、微信信息展示和跳转询盘入口。

7. 留言或询盘入口区
   - 首页放简化表单，字段为姓名、联系方式和需求说明。
   - 完整留言能力放到联系页和留言管理页。

### 6.2 产品分类页

- 展示一级分类产品列表。
- 支持绿茶按采摘时间筛选，支持椴木黑木耳按季节筛选。
- 其他品类按单品列表直接展示。
- 每个产品卡片都应提供进入详情页和立即咨询入口。

### 6.3 产品详情页

- 展示单个产品的主图、详情图和包装图。
- 支持图片切换与点击放大查看。
- 展示产品名称、简介、规格说明、卖点、包装信息和咨询按钮。
- 价格区采用可配置显示方式，默认隐藏。

### 6.4 关于我们页

- 展示店铺介绍、经营历史、种植与生产经验。
- 重点强调长期从业背景和原产地优势。
- 结合产地环境图和生产流程图增强可信度。

### 6.5 联系我们页

- 展示电话联系方式，支持点击直接拨打。
- 展示微信号与二维码或复制提示，支持点击后触发交互反馈。
- 提供完整询盘表单，用于收集客户需求。

### 6.6 留言管理页

- 作为前端演示版内部页面使用。
- 支持查看留言列表、查看详情、标记已回复或未回复。
- 支持填写演示性回复内容，不涉及真实消息发送。

## 7. 产品体系与数据模型

### 7.1 分类体系

一级分类：

- 绿茶
- 椴木黑木耳
- 葛根粉
- 橡子粉
- 茶油
- 农家土鸡蛋

二级分类：

- 绿茶：清明前、清明、谷雨
- 椴木黑木耳：春耳、冬耳

### 7.2 产品数据字段

每个产品至少包含以下字段：

- `id`
- `name`
- `category`
- `subcategory`
- `summary`
- `description`
- `specifications`
- `sellingPoints`
- `images`
- `packageImages`
- `price`
- `showPrice`
- `featured`
- `hero`

### 7.3 其他配置数据

- 轮播内容配置
- 联系方式配置
- 关于我们内容配置
- 全局价格显示开关
- 留言与询盘默认演示数据

整体采用配置驱动方式，使新增产品、替换图片和修改文案不依赖页面结构重写。

## 8. 功能设计

### 8.1 轮播模块

- 首页轮播支持自动播放与手动切换。
- 提供桌面和移动端不同高度策略，避免小屏设备首屏过高。

### 8.2 产品导航与筛选

- 产品中心按一级分类导航。
- 绿茶和黑木耳支持二级分类筛选。
- 移动端使用横向滚动标签或简化筛选结构，桌面端使用显式标签或侧边栏。

### 8.3 图片预览

- 详情页提供轻量级图片放大查看功能。
- 支持浏览主图与细节图，重点服务于农产品外观、干货品质和包装展示。

### 8.4 价格显示

- 价格位采用统一布局设计。
- 通过配置控制显示或隐藏，默认隐藏。
- 隐藏时保留咨询引导，不出现空白结构断层。

### 8.5 联系方式

- 电话使用 `tel:` 链接。
- 微信采用可复制微信号或展示二维码的方式实现。
- 所有咨询按钮尽量汇聚到统一联系路径，便于用户操作。

### 8.6 询盘表单

字段建议如下：

- 姓名
- 联系方式
- 需求说明
- 意向产品

提交成功后展示明确反馈信息，并在前端演示数据中保留提交记录。

### 8.7 留言表单

字段如下：

- 姓名
- 联系方式
- 留言内容

当前阶段数据保存到浏览器本地存储，用于前台演示和管理页读取。

### 8.8 留言管理

- 查看留言列表
- 查看留言详情
- 标记处理状态
- 填写演示性回复

该功能只服务于展示“可管理”的交互闭环，不承担真实客服系统职责。

## 9. 技术架构

### 9.1 技术栈

- React
- Vite
- React Router

### 9.2 目录组织原则

建议按以下职责拆分：

- 页面目录：负责路由页面组装
- 组件目录：负责复用 UI 和业务模块
- 数据目录：负责产品、联系方式和文案配置
- 工具目录：负责本地存储、格式化与通用函数
- 资源目录：负责图片和静态媒体文件

### 9.3 组件划分

重点拆分为以下独立组件：

- 顶部导航
- 轮播模块
- 分类导航
- 产品卡片
- 产品筛选栏
- 图片画廊与预览层
- 联系方式卡片
- 询盘表单
- 留言表单
- 留言列表与管理面板

组件边界应清晰，内容配置与视觉组件分离，避免 JSX 中散落大量静态文案。

## 10. 响应式策略

采用移动端优先设计。

### 10.1 设备适配原则

- 手机端：单列布局，突出电话、微信和表单提交入口。
- 平板端：双列过渡，保持信息清晰和留白平衡。
- 桌面端：多列网格与图文混排，增强展示感和包装呈现。

### 10.2 关键适配点

- 顶部导航在移动端折叠为菜单按钮。
- 轮播文案在移动端缩短并保持按钮可点击。
- 产品卡片从桌面多列退化为移动端单列。
- 表单控件保证触屏友好尺寸。
- 联系方式区在移动端优先显示电话按钮。

## 11. 数据流与状态管理

### 11.1 静态内容流

- 页面从本地配置文件读取产品、分类、联系方式和品牌内容。
- 首页热门产品与主推内容由标记字段自动筛出，而不是写死在页面中。

### 11.2 表单数据流

- 用户填写询盘或留言表单。
- 前端进行必填校验。
- 校验通过后写入浏览器本地存储。
- 页面反馈提交成功状态。
- 留言管理页从本地存储读取数据并进行演示性处理。

### 11.3 状态管理范围

仅使用页面级和组件级状态处理：

- 当前分类与筛选项
- 当前轮播索引
- 当前预览图片
- 表单输入与提交状态
- 留言处理状态

在当前规模下无需引入重型状态管理工具。

## 12. 错误处理与异常场景

- 表单字段为空时给出明确校验提示。
- 本地存储读取失败时回退为空数据并展示友好提示。
- 图片资源缺失时使用占位样式或默认图。
- 无产品数据时展示空状态说明，不让页面结构断裂。
- 价格隐藏时显示“欢迎咨询获取报价”类替代文案，保持转化连续性。

## 13. 性能与可维护性要求

### 13.1 性能

- 对首页横幅图和列表图做压缩处理。
- 优先控制首屏图片体积。
- 避免引入笨重 UI 组件库。
- 对非首屏图片采用懒加载策略。

### 13.2 可维护性

- 内容尽量配置化。
- 组件职责单一，避免超长文件。
- 关键交互添加必要但简洁的注释。
- 页面和组件命名保持语义一致。

## 14. 测试与验收

### 14.1 关键验收范围

- 首页轮播可自动和手动切换。
- 产品分类与子分类筛选正常工作。
- 产品详情页图片预览可用。
- 价格显示开关生效且布局稳定。
- 电话和微信入口可触发预期交互。
- 留言与询盘表单可提交并保存在本地存储。
- 留言管理页可读取并展示前端演示数据。

### 14.2 响应式验收

至少验证以下视口：

- 手机
- 平板
- 桌面

重点检查导航、轮播、产品卡片、联系方式区和表单区域的布局变化。

## 15. 实施建议

建议按照以下顺序实施：

1. 初始化 React + Vite 项目基础结构。
2. 搭建路由与全局布局。
3. 建立产品与站点配置数据。
4. 实现首页核心模块。
5. 实现分类页与详情页。
6. 实现关于我们和联系我们页面。
7. 实现留言与询盘表单、本地存储和留言管理页。
8. 完成响应式优化、基础测试和内容打磨。

## 16. 设计结论

该方案在不接入真实支付和后端的前提下，能够完整满足产品展示、品牌背书、客户咨询和前端留言演示的需求。整体结构适合当前快速落地，也为后续增加真实接口、后台系统和更多产品分类保留了清晰扩展路径。
