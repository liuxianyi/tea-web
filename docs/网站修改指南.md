# 网站修改指南

本指南用于帮助后续维护人员快速修改站点内容，包括联系方式、产品信息、图片资源、价格开关以及 Cloudflare 部署注意事项。

## 1. 先改哪里

这个项目的日常维护主要集中在以下文件和目录：

- `src/siteData.js`
- `public/images/`
- `README.md`

其中：

- `src/siteData.js` 负责站点文案、联系方式、轮播图、产品数据、包装展示和关于我们内容
- `public/images/` 负责图片资源
- `README.md` 负责部署和基础说明

## 3. 修改价格显示

文件位置：

- `src/siteData.js`

全站价格总开关：

```js
features: {
  showPrices: false,
}
```

说明：

- `false`：全站默认隐藏价格，显示“欢迎咨询获取报价”
- `true`：全站直接显示价格

单个产品也有自己的价格开关：

```js
showPrice: false,
```

建议：

- 如果要统一显示价格，优先改全局 `showPrices`
- 如果只想少量产品显示价格，可保留全局关闭，再单独设置某些产品的 `showPrice: true`

## 5. 修改产品信息

文件位置：

- `src/siteData.js`

普通产品数据都在：

```js
export const products = [
  {
    id: 'p1',
    slug: 'gaoshan-mingqian-lvcha',
    name: '高山明前绿茶',
    category: '绿茶',
    subcategory: '清明前',
    summary: '...',
    description: '...',
    specifications: ['125g / 盒', '250g / 礼盒'],
    sellingPoints: ['头采鲜叶', '嫩香明显'],
    price: '¥298 / 125g',
    showPrice: false,
    featured: true,
    hero: true,
    availablePackaging: ['pack-tea-bag-500g', 'pack-tea-small-tin'],
    images: ['/images/hero-tea.svg'],
    packageImages: ['/images/package-visual.svg'],
  },
]
```

主要字段说明：

- `slug`：产品详情页路径标识，建议不要随意改，改了会影响链接
- `name`：产品名称
- `category`：产品所属分类
- `subcategory`：子分类，例如清明前、清明、谷雨、春耳、冬耳
- `summary`：卡片简介
- `description`：详情页说明
- `specifications`：规格数组
- `sellingPoints`：卖点数组
- `price`：价格文本
- `showPrice`：当前产品是否显示价格
- `featured`：是否出现在首页热门产品区
- `hero`：是否作为首页主推候选
- `availablePackaging`：当前产品可选的包装 slug 列表
- `images`：产品图数组
- `packageImages`：包装图数组

### 新增一个产品

做法：

1. 复制一个现有产品对象
2. 修改 `id`
3. 修改 `slug`
4. 修改产品文案和图片路径

注意：

- `id` 不要重复
- `slug` 不要重复

### 删除一个产品

做法：

- 直接删除 `products` 数组中的对应对象

### 修改首页热门产品

做法：

- 让该产品 `featured: true`
- 不想在首页显示时改为 `featured: false`

### 让产品详情显示某个包装

在普通产品里修改：

```js
availablePackaging: ['pack-tea-bag-500g', 'pack-tea-small-tin']
```

只要这里写上对应包装的 `slug`，产品详情页就会自动显示“可选包装”模块。

## 6. 图片怎么放

图片建议放在：

- `public/images/`

推荐结构：

```txt
public/
  images/
    banners/
    products/
      green-tea/
        mingqian/
        qingming/
        guyu/
      black-fungus/
        spring/
        winter/
      kudzu/
      acorn/
      tea-oil/
      eggs/
    packaging/
      tea/
      fungus/
      gift/
    origin/
    contact/
```

推荐命名规则：

- 全部小写
- 使用中划线 `-`
- 不用空格
- 不用中文
- 不用“最终版”“新建文件夹”“图片1”这类名字

示例：

- `green-tea-mingqian-01.jpg`
- `black-fungus-winter-02.jpg`
- `tea-package-01.jpg`
- `origin-tea-mountain-01.jpg`

### 在数据里引用图片

产品图：

```js
images: [
  '/images/products/green-tea/mingqian/green-tea-mingqian-01.jpg',
  '/images/products/green-tea/mingqian/green-tea-mingqian-02.jpg',
]
```

包装图：

```js
packageImages: [
  '/images/packaging/tea/tea-package-01.jpg',
]
```

注意：

- `images` 的第一张会优先作为卡片图和详情页默认主图
- `packageImages` 只放包装、礼盒、袋装、瓶装等图片

### 图片尺寸要求

为了保证网页打开速度和清晰度，建议按图片用途控制尺寸与体积：

- 首页轮播图：
  - 推荐尺寸：`1600 x 900`
  - 建议比例：`16:9`
  - 单张建议体积：`300KB - 800KB`
- 产品详情主图：
  - 推荐尺寸：`1200 x 900` 或更接近 `4:3`
  - 单张建议体积：`200KB - 600KB`
- 产品缩略图：
  - 可与详情图共用，前端会自动缩放
  - 如果单独制作，推荐宽度不低于 `800px`
- 包装展示图：
  - 推荐尺寸：`1200px` 左右宽度
  - 建议比例：`4:3` 或 `1:1`
- 产地与过程图：
  - 推荐尺寸：`1400px` 左右宽度
  - 横图优先，便于页面排版
- 微信二维码：
  - 推荐尺寸：`600 x 600` 以上
  - 保证边缘留白，避免扫码困难

### 图片格式建议

- 优先使用 `webp`
- 暂时不会转换格式时，可使用压缩后的 `jpg`
- 需要透明背景时再使用 `png`

不建议：

- 直接上传手机原图
- 直接上传相机原图
- 使用超过 `2MB - 3MB` 的网页展示图

### 图片尺寸总表

下面这组尺寸适合当前网站使用，也更利于 iPhone 和 Android 设备上的浏览体验：

- 首页轮播图：
  - 推荐尺寸：`1600 x 900 px`
  - 推荐比例：`16:9`
  - 最低可用：`1280 x 720 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`300KB - 800KB`

- 首页热门产品卡片图：
  - 推荐尺寸：`1200 x 900 px`
  - 推荐比例：`4:3`
  - 最低可用：`800 x 600 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`150KB - 350KB`

- 产品详情主图：
  - 推荐尺寸：`1600 x 1200 px`
  - 推荐比例：`4:3`
  - 最低可用：`1200 x 900 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`300KB - 700KB`

- 产品详情附图：
  - 推荐尺寸：`1200 x 900 px`
  - 推荐比例：`4:3`
  - 最低可用：`1000 x 750 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`200KB - 500KB`

- 包装展示图：
  - 推荐尺寸：`1200 x 1200 px` 或 `1200 x 900 px`
  - 推荐比例：`1:1` 或 `4:3`
  - 最低可用：`800 x 800 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`180KB - 450KB`

- 关于我们 / 产地环境图：
  - 推荐尺寸：`1600 x 1000 px`
  - 推荐比例：`16:10` 或接近横图
  - 最低可用：`1400 x 875 px`
  - 推荐格式：`webp` 或 `jpg`
  - 单张建议体积：`250KB - 700KB`

- 微信二维码：
  - 推荐尺寸：`800 x 800 px`
  - 最低可用：`600 x 600 px`
  - 推荐格式：`png`
  - 单张建议体积：`100KB - 300KB`

- Logo / 小图标：
  - Logo 推荐尺寸：`512 x 512 px`
  - favicon 推荐尺寸：`64 x 64 px` 或 `128 x 128 px`
  - 推荐格式：`svg`、`png`

### 当前网站最推荐的统一素材规格

如果你想后续维护最省事，可以统一按下面这组标准准备素材：

- 首页轮播图：`1600 x 900`
- 产品图：`1200 x 900`
- 包装图：`1200 x 1200`
- 产地图：`1600 x 1000`
- 微信二维码：`800 x 800`

### 图片使用建议

- 同一个模块尽量统一长宽比
- 产品卡片图尽量都使用 `4:3`
- 轮播图尽量都使用 `16:9`
- 不建议拿很高的竖图直接做首页主图
- 不建议把一张原始大图同时用于所有场景

### 体积控制建议

- 轮播图最好不要超过 `800KB`
- 普通产品图最好不要超过 `500KB`
- 包装图尽量控制在 `450KB` 内
- 如果单张图片超过 `2MB`，上线前建议先压缩

## 8. Cloudflare 上线前检查

上线前至少检查下面这些项目：

1. 联系电话是否正确
2. 微信号是否正确
3. 微信二维码是否替换为真实图片
4. 产品图和包装图是否全部替换完成
5. 是否需要显示价格
6. `public/_redirects` 是否存在
7. 本地是否已经执行过：

```bash
npm run build
npm run lint
npm run dev -- --host 0.0.0.0
```

## 9. Cloudflare Pages 构建参数

如果重新创建 Pages 项目，使用以下参数：

- `Framework preset`: `Vite`
- `Build command`: `npm run build`
- `Build output directory`: `dist`

项目里已经包含：

- `public/_redirects`
- `.npmrc`

说明：

- `_redirects` 用来解决 React Router 刷新子页面 404
- `.npmrc` 用来固定公共 npm 源，避免构建时走私有镜像导致安装失败

## 10. 当前留言功能说明

目前网站中的留言与询盘功能是前端演示版：

- 数据保存在浏览器本地
- 管理页也是读取本地浏览器数据
- 正式上线后，网站管理员不会自动收到客户信息

如果要正式接单，建议下一步改为：

- Cloudflare Workers 收表单
- 邮件通知
- 或第三方表单服务

## 11. 推荐维护流程

每次修改建议按这个顺序：

1. 改 `src/siteData.js`
2. 替换 `public/images/` 中的资源
3. 本地打开页面检查显示效果
4. 运行：

```bash
npm run build
npm run lint
```

1. 确认无误后提交并部署

## 12. 常见问题

### 刷新产品详情页出现 404

检查：

- `public/_redirects` 是否存在

内容应为：

```txt
/* /index.html 200
```

### 改了图片但页面没变

可能原因：

- 路径写错
- 文件名大小写不一致
- 浏览器缓存未刷新

建议：

- 检查路径
- 强制刷新浏览器
- 确认图片确实已经放到 `public/images/` 下

### 客户留言后我为什么收不到

因为当前只是前端演示版，本地存储不会把数据发给你。
